# MAIN and MAIN.1 System Audit

Status: full-system audit for alpha planning.

This is a planning document. It classifies current checked-in docs/source/tests/issues and
does not implement features.

## Audit Key

- Real: implemented and covered for its stated scope.
- MVP: real but narrow.
- Skeleton: structural placeholder with explicit non-goals.
- Fake/deferred: intentionally unavailable, should fail or be documented clearly.
- Overclaimed: docs/issues imply more than the code proves.

## Cross-Cutting Conclusions

- MAIN should not recreate EPIC-21 through EPIC-26 implementation tasks.
- MAIN should focus on final end-to-end verification, stale-doc correction, and issue
  cleanup readiness.
- MAIN.1 is the real hardening roadmap: resource handles, capabilities, diagnostics,
  source-input handoff, JS-native data bridge, conformance, provider hardening, CLI
  hardening, package/CI hardening, and public alpha docs.
- Public alpha docs are conceptually premature until MAIN.1 gates pass or explicitly defer
  non-working examples.
